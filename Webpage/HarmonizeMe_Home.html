About HarmonizeMe

HarmonizeMe is a Python-based web application that automatically generates chordal harmony to accompany a user's sung melodic input. The user first sings the desired key of their melody, and then sings a short melodic line in a major key. The user then selects the chord that they would like to harmonize each note with. The user is limited to I, ii6, IV, and V chords in major keys only at this point in time. For example, if the user selected the key of C major and the first pitch they sang was an 'F', they would have the option of creating a IV or ii6 chord to accompany their sung pitch. We then use pitch shifting to essentially harmonize their voice with itself and create a 3 voice harmony.

HarmonizeMe could be used as an interesting tool for composers to quickly hear how various harmonies sound without actually needing multiple singers present. It is also simply a fun application for users who are unfamiliar with choral harmony to play around with.

To test our product, we will compare HarmonizeMe to an existing product: Harmony Voice. Harmony Voice automatically harmonizes a user's sung melody, among many other features. We will objectively quantify success based on whether or not the correct pitches are detected and harmonized. We also anticpate some subjective testing in which a musician with a trained ear is asked to determine how 'good' the generated harmony sounds based on the original sung melody.

Progress: We are currently using Aubio's implementation of a pitch tracker, which utilizes the Yin algorithm. Aubio can track both pitch and beat onsets, and we use a Python interface written in C. We run our application in terminal. A demo can be found in the Demo section of the website. Future steps would be to implement a web application through Flask and Boostrap. Additionally, we could work on implementing the Web Audio Toolkit, the first step being to record a signal from a web browser and write it to the disk. We have written the code to convert the user-chosen progression to the pitch-shifted versions of the melody. The pitch tracking is not extremely accurate. Apart from octave errors (which don't affect our results, since we work in scale degrees by modular arithmetic), the pitch tracker also simply returns the wrong pitch. The onset detection errors are not destructive to creating the chords, they just cause confusion when the program asks the user for a chord for a note they never sang. Both these aspects can be improved upon, or just clarified to the user in real-time in the future.
